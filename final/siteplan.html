Site Plan

Corlett Quality Kitchens and Home - Interior Design Company

1. Site Name

Corlett Quality Kitchens and Home

Reason: This name clearly conveys the company’s core focus on custom, high-end kitchen designs and overall home interior solutions. “Corlett Quality” signals both the founder’s name and a commitment to craftsmanship, while “Kitchens and Home” highlights the range of services.

2. Site Purpose

The website will serve as the primary online presence for Corlett Quality Kitchens and Home, focusing on:

  • Showcasing premium interior design services, especially custom kitchen projects and complete home makeovers.
  • Providing a visually rich portfolio (photo galleries) of completed projects to demonstrate craftsmanship and inspire potential clients.
  • Offering detailed service descriptions (interior design, custom cabinetry, color consultation, etc.) so visitors understand precisely how Corlett can transform their spaces.
  • Enabling visitors to request a quote or book a consultation via an embedded contact form for lead generation.

3. Scenarios

Below are two sample questions representing the target audience’s needs. These will guide content and layout decisions:

  • Scenario 1: “I want a modern kitchen with an island and custom cabinetry that maximizes storage—how quickly can I get a quote and see previous examples of similar work?”
  • Scenario 2: “I’m renovating my entire house; I need inspiration for living room color schemes and furniture layouts—where can I view past projects and learn about your design process?”

4. Color Schema

We have selected the following CSS custom properties (variables) to maintain cohesive branding: 

:root {
  --header-bg: #000000;       /* Black – used for header/nav background & footer text */
  --accent-color: #c59d5f;    /* Warm gold – used for headings, buttons, links */
  --card-bg: #f4f4f4;         /* Light gray – used for section backgrounds, portfolio cards */
  --btn-bg: #c59d5f;          /* Same as accent – used for buttons and CTAs */
  --btn-text: #ffffff;        /* White – used for text on buttons */
  --input-border: #cccccc;    /* Neutral gray – used for form input borders */
  --footer-bg: #f4f4f4;       /* Light gray – used for footer background */
}

Where each color will appear:

  • --header-bg (Black) → Header background, navigation bar, footer text.
  • --accent-color (Gold) → All h1/h2/h3 headings, primary buttons (e.g., “Request a Quote”), and any accent elements (e.g., hover states for links).
  • --card-bg (Light Gray) → Background for service cards, portfolio thumbnails, and any “card”-style containers.
  • --btn-bg (Gold) → Button backgrounds (same as accent color for consistency).
  • --btn-text (White on Gold) → Text inside buttons (e.g., “Submit”, “Learn More”).
  • --input-border (Gray Border) → Borders for all form inputs/textareas.
  • --footer-bg (Light Gray) → Footer’s background color to visually separate it from main content.

7. Site Structure (Three Pages)

This website will consist of exactly three top‐level pages. Navigation will be consistent across all pages, and the color schema/typography will remain the same.

  • Home
    – Hero section (full‐width image + overlay text/button)
    – Intro “About Us” snippet
    – Services overview (stacked list on desktop/mobile)
    – Featured portfolio thumbnails (stacked list on desktop/mobile)
    – Quick link to “Request a Quote” (anchor to contact)
    – Footer
  • Services
    – Full descriptions of each service (Interior Design, Home Renovations, Custom Cabinetry, Color Consultation)
    – Individual service images (one per section)
    – “Get a Quote” button that anchors down to the contact form on this page
    – Footer
  • Contact
    – Contact form (Name, Email, Phone, Message, Submit button)
    – Company address and map embed (if desired)
    – Brief testimonial or trust indicators (e.g., star ratings or short quotes)
    – Footer

8. Testing

The following checks will be performed before launch:

  • HTML & CSS Validation: Run through W3C Validator to confirm no syntax errors and proper <meta> tags/programmatic accessibility.
  • Accessibility: Use Lighthouse (Chrome DevTools) to verify color contrast (black text on white background, gold accents on white) and keyboard‐tab navigation through links and form.
  • Responsive Layout: Resize the viewport to ensure:
    • Mobile: All major sections stack vertically, navigation collapses into a hamburger icon, wireframe reflects list style.
    • Desktop: All sections stack vertically in a single column (list view), navigation remains horizontal, wireframe reflects sequence: Header → Hero → About → Services → Portfolio → Contact → Footer.
  • Performance: Check Lighthouse for LCP (<1.2 s), CLS (<0.1), and ensure no render‐blocking CSS. Optimize images (use webp or properly sized JPEGs).
  • SEO & Metadata: Confirm each page has a unique <title> and <meta name="description">, plus Open Graph tags for social sharing.
  • Cross‐Browser Testing: Verify layout and functionality in: Chrome, Firefox, Edge, and Safari (latest stable versions).
